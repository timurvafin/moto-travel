# Moto Travel — планирование мото-путешествий

## О проекте

Планирование мото-путешествий для двух райдеров. Каждая поездка — отдельный каталог в `trips/` с маршрутом, чеклистом и бюджетом.

## Структура

```
moto-travel/
├── CLAUDE.md
├── trip-template.md          # Шаблон маршрута
├── checklist-template.md     # Шаблон чеклиста
├── budget-template.md        # Шаблон бюджета
├── practical-template.md     # Шаблон практической информации (ПДД, валюта, контакты)
└── trips/
    └── <год-название>/
        ├── <год-название>-v<N>.md        # Маршрут, POI
        ├── <год-название>-v<N>.pdf       # PDF-версия маршрута
        ├── <год-название>-practical.md   # Практическая информация (ПДД, валюта, контакты, риски)
        ├── <год-название>.kml            # KML для Google Earth / Google My Maps
        ├── <год-название>-nav.kml        # Навигационный KML для GuruMaps (точки по дням)
        ├── <Название поездки>.kml        # Экспорт из Google My Maps (образец стилей)
        ├── <год-название>-checklist.md   # Чеклист: подготовка + сбор
        ├── <год-название>-budget.md      # Бюджет, жильё и расчёт бензина
        └── bookings/                     # Билеты и брони (PDF)
```

## Как работать с проектом

### Создание новой поездки
1. Создать каталог и скопировать шаблоны:
   ```
   mkdir -p trips/<год-название>
   cp trip-template.md trips/<год-название>/<год-название>-v1.md
   cp checklist-template.md trips/<год-название>/<год-название>-checklist.md
   cp budget-template.md trips/<год-название>/<год-название>-budget.md
   cp practical-template.md trips/<год-название>/<год-название>-practical.md
   ```
2. Именование каталогов: `<год>-<маршрут>`, например `2026-kavkaz-georgia-armenia`
3. Заполнять в порядке: обзор → маршрут по дням (с POI) → жильё
4. Бюджет: заполнить расходы до и в поездке, рассчитать бензин по участкам
5. Чеклист: заполнить секцию «Подготовка к поездке» конкретными задачами из маршрута (визы, бронирования, погранпереходы и т.д.)

### Версионирование
- При каждом существенном обновлении маршрута — увеличивать номер версии и обновлять дату в строке `> Версия: vN (YYYY-MM-DD)`, а также в имени файла
- Имена файлов (`.md` и `.pdf`) содержат суффикс версии через минус: `<название>-v<N>.md`, `<название>-v<N>.pdf`

### Бронирования и билеты
- Хранить в `bookings/` внутри папки поездки
- Именование с датой в начале для сортировки: `<даты>-<тип>-<город/направление>-<название>.pdf` (например `03may-train-kazan-rostov.pdf`, `08-10may-hotel-dombay-gorsky-dom.pdf`)
- В маршруте и бюджете — ссылки на файлы: `[Название](bookings/filename.pdf)`

### При планировании маршрута
- Сначала искать отчёты на источниках из секции "Источники информации"
- Для свежей полевой информации (границы, платежи, состояние дорог, работа POI) — проверять Telegram-чаты с фильтром по дате сообщений
- Все расстояния и время проверять по Яндекс Картам
- Каждый факт подкреплять ссылкой на источник

## Райдеры и мотоциклы

### Мотоцикл 1: Harley-Davidson Ultra Limited

- **Бак:** 22.7 л
- **Расход:** ~6.5 л/100 км
- **Запас хода:** ~340 км
- **Снаряжённая масса:** ~400 кг
- **Привод:** ременной

**Особенности на горных дорогах:**
- Ограниченная манёвренность на серпантинах из-за веса и габаритов
- Осторожность на гравийных/грунтовых участках
- Риск перегрева в пробках и на медленных подъёмах
- Широкий разворот — учитывать ширину дороги
- При планировании горных участков указывать сложность отдельно

### Мотоцикл 2: Yamaha Tracer 9 GT+

- **Бак:** 18 л
- **Расход:** ~5.0–5.5 л/100 км
- **Запас хода:** ~330 км
- **Снаряжённая масса:** ~220 кг
- **Привод:** цепной

## Правила планирования

### Дневные перегоны
- Выезд ~8:00, прибытие 15:00–16:00
- Без вечерних и ночных перегонов
- В шапке дня указывать суммарное время на остановки: `- **Остановки:** ~X ч (перечень)`
- Туристические дни (с остановками и осмотром): не более 400–500 км/день
- Перегоны (транзитные дни без длительных остановок): допустимо до 700–800 км/день при выезде в 06:00 и прибытии до 16:00

### Заправки
- Планировать по мотоциклу с **меньшим** запасом хода (Yamaha, ~330 км)
- Не допускать участков без АЗС длиннее 240 км

### Погода и сезонность
- Проверять прогнозы и климатические данные для региона
- Оценивать целесообразность поездки в выбранный период
- Учитывать высотные перепады (в горах холоднее на 6°C / 1000 м)

### Точки интереса
- Для каждой точки указывать время на осмотр
- Описание: 2-3 предложения с историческим/культурным контекстом (не просто «храм XII в.», а почему он важен)
- При добавлении POI — искать через Google Maps Places API (`maps_search_places` с `locationBias`) для получения Place ID и часов работы
- Ссылка на Google Maps: `- Google Maps: [Название](https://www.google.com/maps/search/?api=1&query=Название&query_place_id=PLACE_ID)`
- ⚠️ Формат `?q=place_id:` НЕ работает в мобильном приложении Google Maps — использовать `?api=1&query=NAME&query_place_id=ID`
- Ссылка на русскую Wikipedia: `- Wikipedia: [Название](https://ru.wikipedia.org/wiki/Название)`
- `⚠️ Часы работы:` — добавлять **только** если есть реальный риск не попасть (закрыто в определённые дни, закрывается рано, сезонный режим). Если работает без ограничений — строку не писать
- Строку «Задержаться на 2+ дня: да» добавлять **только** если место действительно стоит задержки. Если нет — строку не писать вообще
- Добавлять ссылки на использованные отчёты и источники
- Порядок полей POI: Координаты → ⚠️ Часы работы (если есть ограничения) → Вход/Стоимость → Время на осмотр → (Задержаться — только если да) → Описание → Что посмотреть → Фото-точки → Google Maps → Wikipedia → Источники

### Todoist — офлайн-путеводитель

Каждая поездка дублируется в Todoist как офлайн-путеводитель для телефона. Todoist — основной источник информации в дороге вместо markdown-документа.

**Структура проекта:**
```
Проект: Название поездки (цвет: orange)
├── ℹ️ Справочник              — некомплитируемые задачи (isUncompletable)
│   ├── ПДД и штрафы            с полной информацией по странам
│   ├── Деньги и валюта
│   ├── Экстренные контакты
│   ├── Погранпереходы
│   └── Мотоциклы (ТТХ, запас хода)
├── 📋 Общая подготовка         — задачи с дедлайнами
├── 🔧 Подготовка мотоциклов    — заголовки по мотоциклам (isUncompletable), подзадачи ТО
├── 🎒 Сборы Райдера 1          — заголовки-категории (isUncompletable): Экипировка, Личные вещи, Гаджеты, Документы, Снаряжение мото
├── 🎒 Сборы Райдера 2          — аналогично
└── 🗺 Фаза N: Название         — секции маршрута по фазам
    ├── Город → Город — NNN км  — задача-день (dueString = дата + время выезда)
    │   ├── Заправка (15 мин)   — подзадача (dueString = время прибытия, duration)
    │   ├── POI (1.5 ч)         — подзадача с полным описанием
    │   ├── Погранпереход (~1 ч) — подзадача
    │   └── Ночёвка: Город      — подзадача (время прибытия)
```

**Правила оформления:**
- Названия задач: чистый текст без эмодзи (кроме ⚠️ для предупреждений о часах работы)
- Эмодзи только в названиях секций (ℹ️📋🔧🎒🗺)
- Не дублировать дату/номер дня в названии задачи — дата уже в свойствах (dueString)
- Приоритет p1 (красный) — критические остановки: заправки, погранпереходы
- Некомплитируемые задачи (isUncompletable) — для справочников и заголовков-категорий

**Описание задачи-дня:**
```
📏 250 км | ⏱ 9 ч | 🚀 07:00 → ~16:00

**Маршрут:** [Город А → Город Б → ... → Город N](https://www.google.com/maps/dir/lat,lon/.../lat,lon)

Доп. информация (горные участки, план Б...)

**Ночёвка:** Город — Отель
```
- ⏱ включает остановки (= время от выезда до прибытия)
- Маршрут — кликабельная ссылка на Google Maps с ВСЕМИ промежуточными точками
- URL маршрута берётся из основного документа (v2.md)

**Описание подзадачи POI:**
```
Координаты: lat, lon
⚠️ Часы работы: ... (только если есть ограничения)
Вход: стоимость
Время на осмотр: ~N ч

3–4 абзаца с историей, фактами, легендами

Что посмотреть: ...
Фото-точки: ...

[Google Maps](https://www.google.com/maps/search/?api=1&query=Название&query_place_id=ID)
[Wikipedia](https://ru.wikipedia.org/wiki/Название)
```

**Подзадачи — время и длительность:**
- Каждая подзадача имеет dueString (время прибытия) и duration
- Заправки: duration 15m
- Погранпереходы: по оценке (40m–1h)
- POI: по времени осмотра
- Расчёт: от выезда дня + время езды + время предыдущих остановок

**Ссылки Google Maps:**
- POI: `https://www.google.com/maps/search/?api=1&query=Название&query_place_id=PLACE_ID`
- ⚠️ Формат `?q=place_id:` НЕ работает в мобильном приложении
- Маршруты: `https://www.google.com/maps/dir/lat,lon/lat,lon/...`

**Синхронизация:**
- При обновлении маршрута (v2 → v3) — обновить Todoist
- При обновлении Todoist — синхронизировать обратно в v2.md
- Todoist MCP не поддерживает загрузку файлов — PDF бронирований прикреплять вручную

- Если инфа старее порога — **не использовать как факт**, перепроверить: свежий вопрос в чате / официальный источник / актуальный отчёт

## Подводные камни

### Общее
- При планировании поездки за рубеж — проверять требования к сроку действия загранпаспорта для каждой страны на маршруте (у разных стран разные требования: от «действителен на момент пребывания» до «6 месяцев после выезда»)
- При обновлении маршрута (v2 → v3) — обновить также KML (`/generate-kml`), PDF, сводную таблицу дней и Todoist
- Время дня (⏱) должно включать остановки, а не только чистое время в пути
- При сдвиге дат поездки обновлять: (1) обзор (даты, продолжительность, бюджет), (2) сводную таблицу дней, (3) заголовки `### День N`, (4) даты в URL Яндекс Карт (`routes[timeDependent][time]`), (5) таблицу жилья в бюджете
- Формат URL Яндекс Карт: `rtext=lat,lon~lat,lon&rtt=auto&routes[avoid]=unpaved,poor_condition&routes[timeDependent][time]=YYYY-MM-DDTHH:MM:SS`
- Wikipedia URL проверять перед добавлением — статьи могут называться иначе, чем ожидаешь (например, `Бодбийский_монастырь`, а не `Бодбе`)
- Google Maps Places API: искать на русском языке, не на английском
- Координаты POI дублируются в 3 местах: (1) строка `Координаты:` в POI, (2) ссылки на Яндекс/Google Карты дня, (3) KML файл — при изменении обновлять все три
- При переносе POI между днями — обновлять даты в Яндекс Картах И координаты в Google Maps
- Google Places API может вернуть координаты не той точки (например, верхняя станция канатки вместо нижней) — сверять название и адрес в ответе
- При поездках за рубеж проверять местные правила ПДД: камеры, штрафы, ограничения для иностранных номеров — могут не выпустить из страны до оплаты
- Информация из Telegram-чатов без даты = ненадёжна; всегда проверять, когда сообщение написано, и сверять возраст с порогом свежести по теме
- Для быстро меняющихся вопросов (платежи, граница, визы) данные старше 3 месяцев считать устаревшими и перепроверять

### KML
- Координаты: `lon,lat,elevation` — не перепутать с обычным `lat,lon`
- Названия POI без префиксов дней (не «Д6: Джвари», а «Монастырь Джвари»)
- Description: plain text без HTML тегов, ссылку на Google Maps как `📍 URL`
- Стили в формате Google My Maps: `StyleMap` с normal/highlight, иконка `503-wht-blank_maps.png`, цвет через `<color>` (AABBGGRR). Образец — `Кавказ — Грузия — Армения 2026.kml`
- 4 категории: sight (`icon-1535-817717`), nature (`icon-1720-4E342E`), city (`icon-1546-A52714`), border (`icon-1657-E65100`)

### Nav-KML (GuruMaps)
- Нативные иконки GuruMaps: `http://shared.gurumaps.app/s/img/bookmark/BookmarkStyle_N.png` (0–118)
- Style `id` должен совпадать с именем иконки: `id="BookmarkStyle_0"`, `styleUrl="#BookmarkStyle_0"`
- 4 категории: sight (`BookmarkStyle_0`), nature (`BookmarkStyle_15`), city (`BookmarkStyle_9`), border (`BookmarkStyle_8`)
- Простые `<Style>` без StyleMap и без `<color>`
- Точки по дням в `<Folder>`, нумерация `ДN.M. Название`
